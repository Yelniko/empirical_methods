# Lab_4
1. Назвіть мету регресійного аналізу

Мета регресійного аналізу — це дослідження залежності між змінними та побудова моделі, яка дозволяє:
- описати зв’язок між залежною та незалежними змінними;
- оцінити вплив факторів;
- робити прогнози значень;
- знаходити важливі змінні, що впливають на результат.

2. Наведіть коротку характеристику видів регресійного аналізу

- Лінійна регресія
  Найпростіша модель. Показує лінійну залежність: y = ax + b.

- Множинна лінійна регресія
  Використовує кілька змінних X для прогнозу Y.

- Поліноміальна регресія
  Використовує нелінійні залежності (x², x³ тощо).

- Ridge регресія (L2)
  Додає штраф за великі коефіцієнти, зменшує перенавчання.

- Lasso регресія (L1)
  Може зануляти деякі коефіцієнти, допомагає відбору ознак.

- ElasticNet
  Комбінація L1 і L2 регуляризації.


3. У чому відмінність класів Lasso, Ridge та ElasticNet від класів LassoCV, RidgeCV
та ElasticNetCV бібліотеки sklearn?

- без CV → параметри задає користувач
- з CV → параметри підбирає модель автоматично

# Lab_5

За результатами експерименту найвищу точність класифікації показали Logistic Regression та Random Forest (≈95.8%).
Це свідчить про те, що дані добре розділяються, і навіть лінійна модель здатна ефективно виконувати класифікацію.
Модель Naive Bayes також продемонструвала високий результат (≈94.4%), що підтверджує відносну простоту та структурованість даних.
Decision Tree показало дещо гірший результат (≈91.7%), що може бути пов’язано з обмеженням глибини дерева або схильністю до переобучення.
Найгірші результати отримали KNN (≈69.4%) та SVM (≈68.1%). 
Основною причиною є відсутність масштабування ознак, що критично впливає на ці алгоритми.

1. Визначення емпіричних методів дослідження
Емпіричні методи дослідження — це методи отримання знань на основі практичних спостережень, експериментів, вимірювань та аналізу реальних даних. У програмній інженерії вони застосовуються для дослідження процесів розробки, якості програмного забезпечення та ефективності методів роботи.
2. Кількісні та якісні характеристики процесу розробки ПЗ 

Кількісні характеристики:
- кількість рядків коду;
- кількість помилок;
- час розробки;
- кількість комітів;
- продуктивність команди;
- кількість тестів;
- покриття коду тестами.

Якісні характеристики:
- зрозумілість коду;

- задоволеність користувачів;
- зручність інтерфейсу;
- якість документації.
